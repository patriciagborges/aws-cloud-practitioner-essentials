Alto nível do serviço chamado Amazon Elastic Compute Cloud ou EC2 Você precisa de capacidade computacional bruta para hospedar suas aplicações e fornecer o poder computacional que seu negócio precisa. A AWS tem servidores virtuais, acessados através do EC2. O uso do EC2 para computação é altamente flexivel, Econômico e rápido, quando comparado ao uso de datacenters locais. Com a EC2 a AWS já cuidou da parte difícil. EC2 conceitos básicos: A AWS criou os datacenters; A AWS protegel os datacenter; A AWS já adquiriu os servidores; Os servidores já estão instalados pela AWS; Os servidores estão online e prontos para o uso. Você pode usar a capacidade que precisar, basta solicitar as instancias do EC2 que precisa. Quando terminar, pode encerrar ou parar as instancias do EC2. Pague somente pelo que usar. No EC2, você paga somente pelas instancias que estão em execução. O EC2 é executado nas máquinas host físicas gerenciado pela AWS usando tecnologia de virtualização. Quando você gerar uma instancia do EC2, não está necessariamente tomando um host inteiro para sí. Você está compartilhando um host com várias outras instancias. Também conhecido como máquinas virtuais. E um hipervisor em execução na máquina host é responsável por compartilhar os recursos físicos subjacentesentre as máquinas virtuais. Locação multipla: Compartilhamento de hardware subjacentre entre máquinas virtuais. Você tem controle e flexibilidade sobre a configuração das instancias. Quando você provisiona uma instância do EC2, você consegue: Escolher o sistema operacional (linux, windows); Aplicações empresariais internas; Aplicações Web; Bancos de dados; Software de terceiros. Escalabilidade vertical: Você pode começar com menos espaço de CPU ou mais cpu, conforme a demanda que necessitar. Você também controla os aspectos de redes do EC2: Se o acesso é publico ou privado. Computação como serviço (CaaS): Modelo de negócio da AWS usando a computação como serviço.
